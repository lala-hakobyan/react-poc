# Contributing Guidelines

Thank you for being part of the **React POC Projects** community. To keep this a safe, professional and productive space for everyone, please follow these guidelines.

## Community and Contributions
To help keep our discussions organized, please use the following guide for contributing:

- Report typos, bugs or broken links via the **[Issues page on GitHub](https://github.com/lala-hakobyan/react-poc/issues)**.
- Join the conversation or start a new one in the **[Discussions page](https://github.com/lala-hakobyan/react-poc/discussions)**:
    - **Feedback and Suggestions**: Provide general feedback, suggest new topics and share ideas to help improve the project and keep it up to date.
    - **Q&A**: Ask technical questions about the project features and debugging experiments. Everyone is welcome to participate and answer questions, but please keep discussions focused on these areas. If a question is specifically for the author, please mention `@lala-hakobyan` directly.
    - **Announcements**: Follow official announcements and updates from the author regarding the project features and debugging experiments.

## Code of Conduct
- **Be Professional and Kind**: Treat every contributor with respect. Focus on the technical problem or the content, not the person. Every technical question should be respected and there are no stupid questions.
- **Stay Topic-Oriented**: Keep discussions focused on the project features and debugging experiments. If a question is off-topic, do not leave negative comments. Maintainers will take care of it.
- **No Personal Conversations**: Please avoid off-topic chat or personal discussions. This keeps threads searchable and valuable for everyone.
- **No Harassment**: We have zero tolerance for insults, discrimination or any form of harassment.
- **Respect Privacy**: Do not share sensitive or private information about yourself or your company.

## Best Practices for Posting
1. **Search First**: Before asking a question, check if a similar topic already exists.
2. **Be Specific**: Provide enough context so others can understand and help.
3. **Help Others**: If you see a question you can answer, please jump in.

## Moderation
The maintainers reserve the right to move, edit or remove content that violates these guidelines to maintain a high-quality professional environment.

Thank you for helping us build a high-quality resource!
